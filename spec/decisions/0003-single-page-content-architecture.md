# 0003: 初期のページ構成とコンテンツ管理

- 状態: 採用
- 決定日: 2026-06-13

## 背景

- 初期段階では成果物が少なく、成果物一覧の独立ページを設ける必要性が低い
- 訪問者がトップページだけで主要情報を把握できるようにしたい
- サイトヘッダーのページ遷移と、トップページ内のセクション移動の役割を分けたい
- プロフィール各セクションを、TSXの構造を編集せず更新できるようにしたい
- 見出しが `site.ts` とTSXの両方に書かれ、変更時に表示と目次がずれる状態を避けたい
- プロフィールの「これまでの歩み」は、時期別の小見出しを置ける構造にしたい

## 決定

### ページ構成

初期構成はトップページ1ページとする。
成果物の個別ページと一覧ページは設けず、トップページ内の成果物セクションに集約する。

トップページは次の順で構成する。

1. Heroの自己紹介と外部リンク
2. ページ内目次
3. プロフィール
4. できること
5. 成果物
6. Technical Snapshot

サイトヘッダーはページ遷移用のグローバルナビゲーションとし、ページ内目次とは分離する。
外部リンクから連絡手段へ到達できるため、独立した連絡先セクションは設けない。

成果物は `siteContent.works.sections` の配列で区分し、成果物が増えた場合も小見出し単位で追加できるようにする。
プロフィールの「これまでの歩み」は、`site.ts` 側で時期別の小見出しを持てるようにして、大学時代などを内部で分けられるようにする。

### コンテンツ管理

初期段階では、画面に表示する文章、見出し、ラベル、外部リンクを
`src/content/site.ts` に集約する。
TypeScriptの型と `satisfies` によって、データ形式と必須項目をビルド時に検証する。

TSXはHTML構造、要素ID、CSSクラス、描画処理を担当する。
同じ表示文言を目次とセクション見出しなど複数箇所で使う場合は、`site.ts` 内の同じ値を参照する。

## 採用理由

- 内容の更新箇所が明確になり、一人運用での修正漏れを減らせる
- 目次とセクション見出しの不一致を構造的に防げる
- 現在のコンテンツ量では、Markdownローダーや外部スキーマライブラリを追加する利点が小さい
- 表示内容とReactコンポーネントの責務を分ける練習になる

## 結果と制約

- 文言変更は原則として `site.ts` で行う
- 見た目や文書構造の変更はTSXとCSSで行う
- 成果物が少ない間は、独立した成果物ページを追加しない
- 長文や記事が増え、TypeScriptオブジェクトでの編集性が下がった場合はMarkdown等への分離を再検討する
- 個別ページを追加する場合も、トップページだけで主要情報を把握できる構成を維持する

## 比較した選択肢

### 成果物一覧を独立ページにする

成果物が増えた場合は一覧性を高められるが、現段階ではページ内容が薄くなり、トップページから情報が分散するため採用しない。

### 表示文言をTSXへ直接書く

コンポーネント単体では読みやすいが、同じ文言を目次や他のコンポーネントでも使うと重複し、変更漏れが発生するため採用しない。

### JSON、YAML、Markdownへ分離する

非開発者による編集や長文管理には向くが、現段階ではファイル読み込み、スキーマ検証、型生成などの仕組みが増える。
更新者が一人でコンテンツ量も少ないため、必要になるまで導入を保留する。
